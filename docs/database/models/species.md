# Species

## Overview

Represents an encyclopedia species entry used to power species detail pages.

This model is global (not user-scoped) and stores taxonomy/basic metadata, care guidance sections, and discovery helpers (tags and similar species).

## Fields

| Field            | Type     | Optional | Description                                                                      |
| ---------------- | -------- | -------- | -------------------------------------------------------------------------------- |
| id               | string   | no       | Unique identifier generated by Firebase                                          |
| commonName       | string   | no       | Human-readable common name                                                       |
| scientificName   | string   | no       | Scientific/botanical species name                                                |
| family           | string   | yes      | Botanical family                                                                 |
| description      | string   | yes      | General species description                                                      |
| image            | string   | yes      | Primary image URL                                                                |
| tags             | string[] | yes      | Search/filter tags                                                               |
| otherNames       | string[] | yes      | Alternate common names                                                           |
| type             | string[] | yes      | Category labels (e.g. indoor, tropical)                                          |
| similarSpecies   | string[] | yes      | Related species IDs or names used for suggestions                                |
| characteristics  | object   | yes      | Numeric profile (difficulty, toxicity, maintenance, light, pruning, propagation) |
| watering         | object   | yes      | Care section with `text` and image list                                          |
| light            | object   | yes      | Care section with `text` and image list                                          |
| humidity         | object   | yes      | Care section with `text` and image list                                          |
| temperature      | object   | yes      | Care section with `text` and image list                                          |
| soilAndRepotting | object   | yes      | Care section with `text` and image list                                          |
| fertilizing      | object   | yes      | Care section with `text` and image list                                          |
| pestsAndProblems | object   | yes      | Care section with `text` and image list                                          |
| propagation      | object   | yes      | Care section with `text` and image list                                          |

### Nested Types

`SpeciesCharacteristics`:

- `difficulty: number`
- `toxicity: number`
- `maintenance: number`
- `light: number`
- `pruning: number`
- `propagation: number`

`SpeciesCareSection`:

- `text: string`
- `images: SpeciesImage[]`

`SpeciesImage`:

- `url: string`
- `description: string`

## Relationships

- (future) **Plant → Species** (many-to-one)
  Plants currently store free-text species names, but may reference Species records later.

- **Species ↔ Species** (many-to-many, logical)
  `similarSpecies` stores related entries for recommendation/discovery use cases.

## Firestore Storage

`/species/{speciesId}`

## Constraints and Rules

- `commonName` and `scientificName` should be non-empty strings.
- Missing optional arrays are treated as empty arrays in the app.
- Missing optional care sections are normalized to `{ text: "", images: [] }`.
- Missing optional characteristic scores are normalized to `0` values.
- `image` is optional and may be an empty string.
